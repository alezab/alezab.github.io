    Wstęp - Podstawy łamania haseł

    Rodzaje ataków

    Dictionary Attack (Atak słownikowy): - Używa gotowych list słów (wordlists) - Szybki, ale ograniczony do słów w słowniku - Najczęściej używany jako pierwszy krok

    Brute-force (Atak brutalnej siły): - Sprawdza wszystkie możliwe kombinacje - Używa masek do określenia wzorca hasła - Wolniejszy, ale gwarantuje sukces dla małych przestrzeni kluczy

    Hybrid Attack (Atak hybrydowy): - Łączy słownik z regułami lub maskami - Np. słowo + cyfry na końcu

    Narzędzia

    Hashcat: - Najpopularniejsze narzędzie do łamania haseł - Wspiera GPU i CPU - Obsługuje setki typów hashy - Tryby: dictionary (-a 0), brute-force (-a 3), hybrid (-a 6, -a 7)

    John the Ripper: - Alternatywa dla hashcat - Dobry do prostych zadań - Mniej formatów niż hashcat

    Crunch: - Generator wordlist - Tworzy wszystkie kombinacje dla podanych parametrów - Używa wzorców (templates)

    Typy hashy

    Hash Mode (hashcat) Długość Format Przykład zastosowania
    MD5 0 32 hex 5d41402abc4b2a76b9719d911017c592 Stare systemy, kontrola integralności
    SHA-1 100 40 hex aaf4c61ddcc5e8a2dabede0f3b482cd9aea9434d Git commits, certyfikaty (przestarzałe)
    SHA-256 1400 64 hex e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855 Bitcoin, SSL/TLS
    SHA-512 1700 128 hex bardzo długi Serwery Linux/Unix (shadow)
    bcrypt 3200 zmienna $2b$12$... Hasła aplikacji webowych
    scrypt 8900 zmienna SCRYPT:1024:1:1:... Kryptowaluty, hasła

    Hashcat - podstawowe maski

    Maska Znaczenie Przykład
    ?l lowercase (a-z) ?l?l?l = abc, xyz
    ?u uppercase (A-Z) ?u?u = AB, XY
    ?d digit (0-9) ?d?d?d = 123, 999
    ?s special characters ?s = !, @, #, $
    ?a all printable wszystkie znaki
    ?h hex (0-9a-f) używane dla adresów MAC

    Crunch - wzorce (templates)

    Wzorzec Znaczenie Przykład
    @ lowercase (a-z) @@ = aa, ab, zz
    , uppercase (A-Z) ,, = AA, AB, ZZ
    % digit (0-9) %%% = 000, 123, 999
    ^ special characters ^^ = !!, @#

    Przykładowe komendy

    # Hashcat - dictionary attack
hashcat -a 0 -m 0 hash.txt wordlist.txt --force

# Hashcat - brute-force z maską
hashcat -a 3 -m 0 hash.txt '?l?l?l?l' --force

# Hashcat - custom character sets
hashcat -a 3 -m 0 hash.txt -1 abc -2 468 '?1?2?2' --force

# Crunch - generowanie słownika
crunch 4 4 0123456789 -o digits.txt

# Crunch - z wzorcem
crunch 8 8 -t pass%%%% -o passwords.txt

# Crunch - permutacje
crunch 1 20 -p admin password 123 -o permutations.txt

    Zadanie 4.1 - Przykład MD5 z dictionary attack

    Polecenie

    Pod adresem http://127.0.0.1:4001 działa prosty serwer HTTP udostępniający dwa endpointy: - http://127.0.0.1:4001/hash - pobieranie hashu MD5 - http://127.0.0.1:4001/submit - wysyłanie złamanego hasła

    Zadanie: Złam hash MD5 używając dostępnego słownika i wyślij wynik na serwer.

    Rozwiązanie

    (a) Uruchomienie serwera

    docker run -p 4001:4001 --name ex1 docker.io/mazurkatarzyna/pass-cracking-ex1:latest

    Lub zapasowy: 

    docker run -p 4001:4001 --name ex1 ghcr.io/mazurkatarzynaumcs/pass-cracking-ex1:latest

    (b) Pobranie hashu

    curl -s -X GET http://127.0.0.1:4001/hash | jq .

    {
  "hash": "19005ef9d4e2abbf7b029b61f094a286",
  "message": "Spróbuj złamać ten hash!"
}

    curl -s -X GET http://127.0.0.1:4001/hash | jq -r .hash > hash.txt

    (c) Złamanie hashu hashcat

    hashcat -a 0 -m 0 hash.txt slowniki/ex41/wordlist.txt --force
hashcat -a 0 -m 0 hash.txt --show

    Hasło: kidney81

    (d) Wysłanie wyniku

    curl -s -X POST http://127.0.0.1:4001/submit \
  -H "Content-Type: application/json" \
  -d '{"word": "kidney81"}' | jq .

    Wyjaśnienie: - -a 0 - tryb dictionary attack - -m 0 - typ hashu MD5 - --force - wymusza działanie na CPU (bez GPU)

    # W crunch: % = cyfra (0-9)
crunch 8 8 -t pass%%%% -o dict49.txt

    (d) Złamanie hashu

    Hashcat ze słownikiem: 

    hashcat -a 0 -m 0 hash49.txt dict49.txt --force

    Hasło: pass#### (przykład: pass1234)

    (e) Wysłanie wyniku

    Wyjaśnienie wzorców crunch: - % - cyfra (0-9) - @ - mała litera (a-z) - , - wielka litera (A-Z) - ^ - znak specjalny

    # Flaga -p generuje wszystkie permutacje podanych słów

    Zadanie 4.11 - Wzorzec 5 znaków

    Polecenie

    Pod adresem http://127.0.0.1:4011 działa prosty serwer HTTP udostępniający hash MD5 do złamania.

    Zadanie: Wiedząc, że hasło ma wzorzec: - Znak 1: wielka litera (A-Z) - Znak 2: mała litera (a-z) - Znak 3: cyfra (0-9) - Znaki 4-5: znaki specjalne (!, @, #, $, %, ^, &, *, itd.)

    Wygeneruj słownik i złam hash.

    Rozwiązanie

    (a) Uruchomienie serwera

    docker run -p 4011:4011 --name ex11 docker.io/mazurkatarzyna/pass-cracking-ex11:latest

    (b) Pobranie hashu

    curl -s -X GET http://127.0.0.1:4011/hash | jq -r .hash > hash411.txt

    Hash: 68793f841751bef32d2e000242e782f9

    (c) Generowanie słownika

    # W crunch:
# , = wielka litera (A-Z)
# @ = mała litera (a-z)
# % = cyfra (0-9)
# ^ = znaki specjalne
crunch 5 5 -t ',@%^^' -o dict411.txt

    Słownik zawiera ~7.3 miliona kombinacji

    (d) Złamanie hashu

    Hashcat ze słownikiem: 

    hashcat -a 0 -m 0 hash411.txt dict411.txt --force
hashcat -a 0 -m 0 hash411.txt dict411.txt --show

    Lub hashcat z maską: 

    # ?u = wielka litera, ?l = mała litera, ?d = cyfra, ?s = znaki specjalne
hashcat -a 3 -m 0 hash411.txt '?u?l?d?s?s' --force

    John the Ripper: 

    john --wordlist=dict411.txt --format=raw-md5 hash411.txt
john --show --format=raw-md5 hash411.txt

    Hasło: Xe3*!

    (e) Wysłanie wyniku

    curl -s -X POST http://127.0.0.1:4011/submit \
  -H "Content-Type: application/json" \
  -d '{"word": "Xe3*!"}' | jq .

    Polecenie

    Pod adresem http://127.0.0.1:4012 działa prosty serwer HTTP udostępniający hash SHA-1 do złamania.

    Zadanie: Wiedząc, że hasło składa się z 4 małych liter (a-z), złam hash używając brute-force (bez słownika).

hashcat -a 3 -m 100 hash412.txt '?l?l?l?l' --force

    Wyjaśnienie: - -a 3 - tryb brute-force (mask attack) - -m 100 - SHA-1 - ?l?l?l?l - 4 małe litery - Przestrzeń kluczy: 26^4 = 456,976 kombinacji

    Zadanie 4.14 - Custom character sets

    Polecenie

    Pod adresem http://127.0.0.1:4014 działa prosty serwer HTTP udostępniający hash SHA-1 do złamania.

    Zadanie: Wiedząc, że hasło ma wzorzec: - Znak 1: litera z {a, b, c} - Znak 2: cyfra z {4, 6, 8} - Znak 3: znak specjalny z {*, %, :}

    Złam hash używając custom character sets.

    Rozwiązanie

    (a) Uruchomienie serwera

    docker run -p 4014:4014 --name ex14 docker.io/mazurkatarzyna/pass-cracking-ex14:latest

    Hash: 0b26fa991a9d529f604215255e970a2c2fc971a6

    (c) Złamanie hashu

    Hashcat z custom character sets: 

    # -1 abc = zestaw 1: a, b, c
# -2 468 = zestaw 2: 4, 6, 8
# -3 *%: = zestaw 3: *, %, :
hashcat -a 3 -m 100 hash414.txt -1 abc -2 468 -3 '*%:' '?1?2?3' --force
hashcat -a 3 -m 100 hash414.txt -1 abc -2 468 -3 '*%:' '?1?2?3' --show

    Lub crunch + john: 

    crunch 3 3 -t '@%^' -1 abc -2 468 -3 '*%:' > dict414.txt
john --wordlist=dict414.txt --format=raw-sha1 hash414.txt
john --show --format=raw-sha1 hash414.txt

    (d) Wysłanie wyniku

    curl -s -X POST http://127.0.0.1:4014/submit \
  -H "Content-Type: application/json" \
  -d '{"word": "a4:"}' | jq .

    Wyjaśnienie: - -1, -2, -3 - definiują custom character sets - ?1, ?2, ?3 - odnoszą się do zestawów w masce - Przestrzeń kluczy: 3 × 3 × 3 = 27 kombinacji

    Zadanie 4.15 - Adres MAC

    Polecenie

    Pod adresem http://127.0.0.1:4015 działa prosty serwer HTTP udostępniający hash SHA-1 do złamania.

    Zadanie: Wiedząc, że hasło to adres MAC w formacie: - Prefix: 00:14:22:ff:ff: (stały) - Suffix: XX (2 znaki hex: 0-9, a-f)

    Złam hash używając brute-force.

    Rozwiązanie

    (a) Uruchomienie serwera

    docker run -p 4015:4015 --name ex15 docker.io/mazurkatarzyna/pass-cracking-ex15:latest

    (b) Pobranie hashu

    curl -s -X GET http://127.0.0.1:4015/hash | jq -r .hash > hash415.txt

    Hash: 5540dfe65be6a76560e2b19b3771b38aa44dcd8f

    (c) Złamanie hashu

    Hashcat z custom hex charset: 

    # -1 0123456789abcdef = charset hex
hashcat -a 3 -m 100 hash415.txt -1 0123456789abcdef '00:14:22:ff:ff:?1?1' --force
hashcat -a 3 -m 100 hash415.txt -1 0123456789abcdef '00:14:22:ff:ff:?1?1' --show

    Lub z hashcat hex charset ?h: 

    # ?h = hex (0-9a-f)
hashcat -a 3 -m 100 hash415.txt '00:14:22:ff:ff:?h?h' --force

    Lub crunch + john: 

    crunch 17 17 -t '00:14:22:ff:ff:@@' -1 0123456789abcdef > dict415.txt
john --wordlist=dict415.txt --format=raw-sha1 hash415.txt
john --show --format=raw-sha1 hash415.txt

    Hasło: 00:14:22:ff:ff:d0

    (d) Wysłanie wyniku

    curl -s -X POST http://127.0.0.1:4015/submit \
  -H "Content-Type: application/json" \
  -d '{"word": "00:14:22:ff:ff:d0"}' | jq .

    Wyjaśnienie: - OUI (Organizationally Unique Identifier): 00:14:22 - Dell - Przestrzeń kluczy: 16^2 = 256 kombinacji (00-ff) - ?h w hashcat = hex digit (0-9a-f)

    Hashcat - tryby ataku

    Tryb Nazwa Użycie Przykład
    -a 0 Dictionary Gotowy słownik hashcat -a 0 -m 0 hash.txt wordlist.txt
    -a 3 Brute-force Maski hashcat -a 3 -m 0 hash.txt '?l?l?l?l'
    -a 6 Hybrid (dict+mask) Słownik + cyfry hashcat -a 6 -m 0 hash.txt dict.txt '?d?d'
    -a 7 Hybrid (mask+dict) Cyfry + słownik hashcat -a 7 -m 0 hash.txt '?d?d' dict.txt

    Hashcat - maski (charset built-in)

    Maska Opis Przykład
    ?l Lowercase (a-z) ?l?l?l = abc
    ?u Uppercase (A-Z) ?u?u = AB
    ?d Digit (0-9) ?d?d?d = 123
    ?s Special ?s?s = !@
    ?a All printable ?a?a?a = A2!
    ?h Hex (0-9a-f) ?h?h = a3
    ?b Binary (0x00-0xff) wszystkie bajty

    Crunch - wzorce (templates)

    Wzorzec Opis Przykład
    @ Lowercase (a-z) @@@ = abc
    , Uppercase (A-Z) ,,, = ABC
    % Digit (0-9) %%% = 123
    ^ Special ^^ = !@

    Typy hashy - porównanie

    Hash Bezpieczeństwo Szybkość łamania Zastosowanie
    MD5 ❌ Słabe ⚡ Bardzo szybkie Przestarzałe
    SHA-1 ❌ Słabe ⚡ Bardzo szybkie Git (nie do haseł)
    SHA-256 ✅ Dobre* ⚡ Szybkie Bitcoin, SSL
    SHA-512 ✅ Dobre* ⚡ Szybkie Linux/Unix
    bcrypt ✅✅ Bardzo dobre 🐌 Wolne Web apps
    scrypt ✅✅✅ Najlepsze 🐌🐌 Bardzo wolne Kryptowaluty

    *SHA-2 jest bezpieczny, ale szybki - dla haseł lepiej użyć bcrypt/scrypt

    Najlepsze praktyki

    Generowanie słowników: 1. Używaj crunch dla małych przestrzeni kluczy (< 10 milionów) 2. Dla większych - używaj masek w hashcat 3. Permutacje (-p) dla znanych słów

    Łamanie haseł: 1. Zacznij od dictionary attack (najszybszy) 2. Jeśli nie działa, spróbuj brute-force z maską 3. Dla wolnych hashy (bcrypt/scrypt) używaj małych słowników

    Bezpieczeństwo: - Minimum 12 znaków dla haseł - Mieszaj wielkie, małe, cyfry, znaki specjalne - Używaj bcrypt/scrypt zamiast MD5/SHA-1 - Nigdy nie używaj prostych wzorców
